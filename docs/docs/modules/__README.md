# Adding New Module Documentation

This guide explains how to add documentation for a new module to the Midil Kit documentation site.

## Structure

The modules documentation follows a simple structure:

- **Single-file modules**: If your module has only one documentation file, place it directly in `modules/` as `module-name.md`
- **Multi-file modules**: If your module has multiple related documentation files, create a folder `modules/module-name/` and place all files there

## Adding a Single-File Module

For modules with just one documentation file:

1. Create `docs/docs/modules/your-module.md`
2. Add frontmatter with `sidebar_label`:

```markdown
---
sidebar_position: 6
sidebar_label: 📦 Your Module (midil.your_module)
title: Your Module
description: Brief description of your module
---

# Your Module

Content here...
```

The `sidebar_label` will be displayed in the sidebar. The `sidebar_position` determines the order.

## Adding a Multi-File Module

For modules with multiple related documentation files:

1. Create a folder: `docs/docs/modules/your-module/`
2. Create `_category_.json` in that folder:

```json
{
  "label": "📦 Your Module (midil.your_module)",
  "position": 6,
  "collapsed": false
}
```

3. Add your documentation files to the folder:
   - `index.md` or `overview.md` - Main module documentation
   - `sub-module.md` - Sub-module documentation
   - `another-topic.md` - Additional topics

4. Each file should have frontmatter:

```markdown
---
sidebar_position: 1
title: Module Overview
---

# Module Overview

Content here...
```

## Example: CLI Module

The CLI module is a good example of a multi-file module:

```
modules/
  cli/
    _category_.json       # Defines the category label
    test.md              # Sub-documentation
    commands.md          # Another sub-documentation (if added)
```

## Example: Auth Module

The Auth module is a single-file module:

```
modules/
  auth.md                # Single comprehensive documentation file
```

## Sidebar Configuration

The sidebar is automatically generated from the filesystem structure. No manual configuration needed in `sidebars.js` - it uses:

```javascript
{
  type: 'autogenerated',
  dirName: 'modules',
}
```

This means:
- Single `.md` files in `modules/` appear with their `sidebar_label`
- Folders with `_category_.json` appear as categories with all files inside

## Best Practices

1. **Use descriptive labels**: Include emoji and Python module path in `sidebar_label`
2. **Consistent positioning**: Use `sidebar_position` to maintain logical ordering
3. **Clear folder structure**: Group related documentation together
4. **Index files**: Use `index.md` or `overview.md` as the main entry point for folder-based modules
